Test Page - Report System

📋 Instrucciones de Testing

  1. Abre esta página con: test-page-report.html?applicationId=TEST_FLOW#/page1
  2. Abre el popup de la extensión
  3. Ingresa: test-container
  4. Habilita todos los checks
  5. Click "Run Analysis"
  6. Verifica que encuentra ~10 errores
  7. Cambia la URL a: #/page2
  8. Ejecuta análisis nuevamente
  9. Click "View Details" en el popup
  10. Verifica que hay 2 páginas (page1 y page2)
  11. Click "Download Excel"
  12. Abre el CSV en Excel y verifica el formato

🧪 Tests Adicionales

Test A: Múltiples Páginas

  1. Ejecuta análisis en #/page1 → Guarda N errores
  2. Cambia URL a #/page2
  3. Ejecuta análisis nuevamente
  4. Verifica que el resumen muestra 2 páginas
  5. Verifica que los errores se acumulan (no se reemplazan)

Test B: Re-análisis de Misma Página

  1. Ejecuta análisis en #/page1 → Nota el número de errores
  2. Ejecuta análisis nuevamente (sin cambiar nada)
  3. Verifica que el número de errores es el mismo (no se duplica)

Test C: Múltiples Flujos

  1. Ejecuta análisis con ?applicationId=FLOW_A#/page1
  2. Cambia a ?applicationId=FLOW_B#/page1
  3. Ejecuta análisis
  4. Verifica que el resumen muestra 2 flujos
  5. Descarga Excel y verifica que hay 2 secciones

Test D: Atributos en CSV

  1. Ejecuta análisis completo
  2. Descarga Excel
  3. Verifica que cada fila tiene:
    • FLOW NAME: TEST_FLOW
    • PAGE NAME: page1
    • ERROR CODE: FONT-SIZE, ICON-SIZE, etc.
    • ELEMENT TYPE: button, span, div, etc.
    • ID: los IDs de los elementos
    • TRACK ID: los trackid de los elementos
    • DATA-TEST-ID: los data-testid de los elementos
    • ERROR MESSAGE: mensaje descriptivo

Test E: Clear Report

  1. Acumula errores en varias páginas
  2. Click "Clear Report"
  3. Confirma el diálogo
  4. Verifica que el resumen muestra "No errors collected yet"
  5. Ejecuta análisis nuevamente
  6. Verifica que empieza desde cero
